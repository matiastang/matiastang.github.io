# 介绍

@vuepress/theme-blog是由VuePress (opens new window)维护。在插件中实现所有常用和必要的博客功能，并更加注重主题中的交互体验。

# 安装

$ pnpm add -D @vuepress/theme-blog

# 目录结构

├── blog
│   ├── _posts
│   │   ├── 2018-11-7-frontmatter-in-vuepress.md #example
│   │   ├── 2019-2-26-markdown-slot.md #example
│   │   └── 2019-5-6-writing-a-vuepress-theme.md #example
│   └── .vuepress
│       ├── `components` _(**Optional**)_
│       ├── `public` _(**Optional**)_
│       ├── `styles` _(**Optional**)_
│       │   ├── index.styl
│       │   └── palette.styl
│       ├── config.js
│       └── `enhanceApp.js` _(**Optional**)_
└── package.json
  • .vuepress/config.js: 配置入口文件，也可以是ts、yml或toml.
  • 与普通Vuepress唯一不同的是：帖子内容需要放到_posts文件夹下面。

注意blog目录名称可以是任意的，只要能和package.json中的启动命令对上就行。

# 配置

在.vuepress的配置文件config.ts中配置插件。

import { defineConfig } from 'vuepress/config'

export default defineConfig({
    title: 'Matias Tang',
    theme: '@vuepress/theme-blog',
})

# 添加.md文档

_posts文件夹是您的博客文章所在的位置。

所有博客文章文件都可以以前言开头。title是必需的，其他字段都是可选的，查看frontmatter (opens new window)了解更多详情。

---
title: Hello World
date: 2020-01-11
author: Billyyyyy3320
location: Taipei  
---

> This is official blog theme.

My content.

# 配置package.json

{
	"scripts": {
		"docs:dev": "vuepress dev docs",
		"docs:build": "vuepress build docs",
	},
}
  • 上面提到了blog目录名称可以是任意的，只要能和package.json中的启动命令对上就行。这里就是例子，文件夹命名为docs。

注意 运行pnpm run docs:dev启动服务时，如果报错：Error: error:0308010C:digital envelope routines::unsupported (opens new window)则是node版本的问题，可以点击查看。
