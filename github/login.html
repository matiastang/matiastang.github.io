# GitHub Login

GitHub于2021 年 8 月 13 日起不再支持密码验证登录。有如下两种方式登录：

  1. SSH 免密登录
  2. Token 登录

GitHub提供了HTTPS、SSH、GitHub CLI的方式来拉取或提交代码。

HTTPS的远程地址如：https://github.com/[USER]/[REPO] SSH的远程地址如：git@github.com:[USER]/[REPO]

# 使用Token登录

配置的token就相当于密码，后续需要密码的时候用token替换

进行git push命令行提交的时候，会让你输入Username和Password

git push
Username for 'https://github.com': username
Password for 'https://matiastang@github.com': 你的token

直接把token当做Password输入。

我们知道URI的完整格式为：[scheme]://[user]:[passwd]@[host]:[port][path][?query][#fragment]所以我们可以直接在关联的远程地址中配置上token就能直接完成提交了。

  • 直接修改.git/config文件

.git/config是隐藏文件。需要显示出来，mac 快捷键是control + cmd + .

修改 remote 中的 url 为:

url = https://[TOKEN]@github.com/[USER]/[REPO]

TOKEN是上面申请的token。 USER是用户名。 REPO是仓库名。

  • 通过命令更新
git remote set-url origin https://[TOKEN]@github.com/[USER]/[REPO]
  • 删除现在的远程分支，然后创建新的远程分支。
git remote remove origin
git remote add origin https://[TOKEN]@github.com/[USER]/[REPO]

注意：不建议使用在关联的远程地址中配置token的方式完成登录，公开的项目会泄露token。

在地址中配置token的方式一般在配置GitHub actions (opens new window)中使用，如：在GitHub actions做提交代码到另外一个仓库时，需要使用token的方式完成仓库的认证。但及时是在GitHub actions (opens new window)中使用gitHub tokens (opens new window)也是通过gitHub secrets (opens new window)来使用的。

注意：默认情况下，GitHub Pages 站点在互联网上是公开可用的，即使站点的存储库是私有的或内部的。如果您的项目网站是从使用 GitHub 企业云的组织拥有的私有或内部存储库发布的，则可以管理该网站的访问控制。在具有托管用户的企业中，所有 GitHub Pages 网站都是私下发布的。否则，如果站点的存储库中有敏感数据，则可能需要在发布之前删除这些数据。
