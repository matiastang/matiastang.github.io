# github tokens

# 介绍

使用 GitHub API 或命令行时，可使用 Personal access token 替代密码向 GitHub 进行身份验证。 Personal access token 旨在代表你自己访问 GitHub 资源。

# 生成token

官方步骤 (opens new window)

  1. 登录github账号
  2. 点击进入账号的Setting
  3. 点击Developer settings
  4. 下拉Personal access tokens选择Tokens
  5. 下拉Generage new token选择Generage new token（classic）
  6. 填写名称，随便填写
  7. 选择过期时间
  8. 选择权限，根据需求选择权限
  9. 点击生成token就行了
  10. 记着保存好生成的token，刷新页面就没有了

# 使用

# 1. token当做密码使用

进行git命令行提交的时候，会让你输入Username和Password

git push
Username for 'https://github.com': matiastang
Password for 'https://matiastang@github.com': 你的token

直接把token当做Password输入。

# 2. 直接更改.git/config文件 （不推荐）

注意：不建议使用这种方式使用，会泄露token。

  • 直接修改.git文件

.git/config是隐藏文件。需要显示出来，mac 快捷键是control + cmd + .

修改 remote 中的 url 为:

url = https://[TOKEN]@github.com/[USER]/[REPO]

TOKEN是上面申请的token。 USER是用户名。 REPO是仓库名。

  • 通过命令更新
git remote set-url origin https://[TOKEN]@github.com/[USER]/[REPO]

或：删除现在的远程分支，然后创建新的远程分支。

git remote remove origin
git remote add origin https://[TOKEN]@github.com/[USER]/[REPO]

# 3. 配置CI/CD自动部署时使用token

虽然token等当密码使用，但是一般我们长期使用都是添加公钥到仓库的，这样在本机上就能完成身份认证了。 虽然通过更改.git/config中的远程地址，达到了使用token关联到了仓库，但是上面也说了会有暴露的风险。

token的主要使用地方还是在配置CI/CD自动部署时。

如下是github actions (opens new window)的一个.yml配置，里面就使用了token。只是为了避免token暴露，通过配置项目的secrets (opens new window)来访问配置的token的。

name: vueperss deploy

on: [push]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
    - name: checkout
      uses: actions/checkout@master

    - name: vueperss build and deploy
      run: |
        yarn install
        yarn docs:build
        cd docs/.vuepress/dist
        git init
        git config --global user.email ${{ secrets.MT_EMAIL }}
        git config --global user.name ${{ secrets.MT_NAME }}
        git add -A
        git commit -m 'Auto deploy from Github Actions'
        git push -f https://${{ secrets.MT_TOKEN }}@github.com/matiastang/matiastang.github.io master
