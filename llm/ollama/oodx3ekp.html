Ollama 使用

约 4469 字大约 15 分钟

2025-02-07

Ollama 是一个专注于本地化部署和运行大型语言模型（LLM）的工具，旨在让用户能够在自己的设备上高效地运行和微调模型。

官方首页简单介绍了 Ollama 能在本地运行的模型，包括 Llama 3.3, DeepSeek-R1, Phi-4, Mistral, Gemma 2 等。现在DeepSeek-R1很火!

下载

直接在ollama download下载对应系统的版本即可。

安装

下载完成之后，直接解压后放入应用程序中即可。

命令行查看版本

$ ollama --version
ollama version is 0.5.7

使用

在ollama Models查找到对应的模型，使用ollama run ***即可。

**1. Unicode 的定义：**
   Unicode 是一种用于表示所有字符的编码标准，涵盖从1551年提出的国际标准化组织（ISO）制定的标准。它允许将大量不同语言、文化背景下的符号统一编码，并为它们
提供一个公共的概念空间。

**2. UTF-8 的定义：**
   UTF-8（Unicode Transformation Format 8-bit字节）是 Unicode 标识符的一种编码方式，广泛用于计算机网络中传输文本信息。UTF-8 将大部分 Unicode 字符编码为
1到4个字节的值，并且在不同的系统间兼容。

**3. UTF-8 和 Unicode 的关系：**
   - **编码方式相同:** UTF-8 是 Unicode 标识符的一种编码方式，也就是说，它可以用来表示和解读 Unicode 的字符。
   - **兼容性:** 在不同编程语言、设备上进行编码转换时，UTF-8 能够正确地将 Unicode 字符转换为所需的格式（如 ASCII、Unicode 等）。

**4. UTF-8 适用于哪些内容？**
   - **文本传输和存储:** 在网页服务器等网络环境中，使用 UTF-8 解码和处理文本信息。
   - **文件系统和数据库管理:** 将 Unicode 字符存入文件系数组时，需要转换为 UTF-8 的编码以便文件系统的兼容性。

**5. 什么是编码转换？**
   编程语言（如 Java、Python）等程序通常在运行时将输入数据从字符类型转换为特定的数据格式。例如，在生成 HTML 页面的代码后，需要使用 CSS 和 JavaScript 进行
编码转换以实现输出页面。

**6. UTF-8 的特点：**
   - **多字节编码:** 大多数 Unicode 字符仅占用1到4个字节的空间。
   - **系统兼容性:** 支持广泛不同的操作系统和编程语言进行编码和解码。

**7. 为什么使用 UTF-8？**
   - **支持广泛字符集:** UTF-8 完全涵盖了 Unicode 的范围，使文本传输更通用且灵活。
   - **兼容性和效率:** 在不同系统之间兼容，并且在实际应用中表现良好，适合各种网络和计算机环境。

**总结：**
UTF-8 是 Unicode 标识符的一种编码方式，在处理文本信息时广泛使用。它通过多字节编码提供了一种兼容性高、适用范围广的解决方案。

Ollama使用显存还是内存？

Ollama 在本地部署大模型时优先使用显存（VRAM），当显存不足时，会将超出的部分使用主机内存（RAM），但这可能会影响性能。

Ollama 的显存与内存使用策略

  1. 优先使用 GPU 显存：
  • 如果你的设备有 NVIDIA GPU（CUDA） 或 Apple Silicon（Metal），Ollama 会首先使用 GPU 显存进行推理。
  • 显存足够时，推理速度会很快，适用于高效运行大模型（如 LLaMA 3 8B、Mistral 7B）。
  1. 显存不足时，部分数据会溢出到 RAM：
  • 如果显存不够，Ollama 会将部分计算或数据缓存在 CPU 内存（RAM）。
  • 这通常会导致推理速度变慢，因为 CPU 内存带宽比显存低，而且 GPU 与 RAM 之间的数据传输存在瓶颈。
  • 在 Windows 和 Linux 上，可能会使用 Swap 交换分区，导致进一步的性能下降。
  1. 纯 CPU 模式（无 GPU）：
  • 如果你的设备没有 GPU 或者 显存完全不足，Ollama 会完全依赖 CPU，并使用 RAM 进行计算。
  • 这会导致推理速度大幅降低，特别是当使用 13B 或更大的模型时。

Ollama使用显存还是内存？取决于模型的大小和显存的大小。如果模型的大小小于显存的大小，那么Ollama会使用显存来加速计算。如果模型的大小大于显存的大小，那么Ollama会使用内存来加速计算。

这么看来Mac的统一内存是比较占优的，虽然也很贵！

如何优化 Ollama 的内存管理？

如果你的显存不够，可以尝试以下方法进行优化：

  1. 使用更小的模型
  • 量化模型：Ollama 默认使用 GGUF 量化模型（如 llama3:8b）。
  • 你可以选择 更低精度的模型，例如 7B、4B 或 Q4_0 量化版本，以减少显存占用
  1. 调整 GPU 使用策略：

Mac 上可以使用 OLLAMA_METAL=1 变量强制使用 Metal 加速：

OLLAMA_METAL=1 ollama run llama3

Windows/Linux 上，可以使用 CUDA_VISIBLE_DEVICES 限制 GPU 资源

CUDA_VISIBLE_DEVICES=0 ollama run llama3
  1. 限制 RAM 使用：
  • 可以使用 ollama run --num-ctx 2048 限制上下文长度，减少内存消耗
ollama run llama3 --num-ctx 2048

默认情况下，大模型可能会使用 4096 或更高的 context length，导致更多显存占用。
